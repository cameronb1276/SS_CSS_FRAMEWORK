SS CSS Framework

A lightweight, class-based CSS framework for business websites, dashboards, SaaS pages, admin panels, and client projects.

View demo Open minified CSS

Single-file CSS

Link one stylesheet and attach predictable ss-* classes to the elements you want to style.

  <main class="ss-container ss-py-8">
    <h1 class="ss-h1">Page title</h1>
    <button class="ss-btn">Action</button>
  </main>
</body>

What Is Included

Foundation

Tokens, typography, media, forms, tables, code, blockquotes, and content classes.

Utilities

Containers, grids, flex, display, spacing, sizing, text, color, borders, radius, shadows, position, and z-index.

Components

Buttons, badges, alerts, cards, forms, input groups, tables, navs, dropdowns, modals, toasts, progress, skeletons, accordions, lists, and avatars.

Containers and Grids

Containers

  • .ss-container for standard pages.
  • .ss-container-narrow for readable text pages.
  • .ss-container-wide for dashboards.
  • .ss-container-full for app shells.

Dark mode

Dark Mode

Dark mode is opt-in and class-based. Use .ss-theme-dark, [data-ss-theme="dark"], or .ss-theme-system.

<body class="ss-page ss-theme-dark">
  ...
</body>

Dark card

Surfaces, borders, focus rings, status colors, overlays, and shadows adapt through tokens.

Accessibility and Browser Support

Accessibility

Native semantics are preserved. Focus states are visible, links remain underlined, disabled states remain readable, validation states use color plus a thicker left border, and reduced motion is respected.

Browser Support

The framework targets current Chromium-based browsers, Firefox, and Safari. No legacy Internet Explorer support is planned.

Customization

:root {
  --ss-color-primary: #0ea5e9;
  --ss-radius-default: 0.375rem;
  --ss-page-gutter: clamp(1rem, 4vw, 2.5rem);
}
